Základní tok událostí

  1. Systém provede validaci na vstupu získaných hodnot (viz VAL01: Kontrola vstupních dat).
  2. Systém vytvoří novou instanci entity Fotografie stanoviště, provede následující akce:
    1. přiřadí jedinečné identifikační číslo,
    2. nastaví hodnoty atributů dle informací získaných na vstupu (vstupní parametr Zakládaná fotografie stanoviště),
    3. nastaví hodnotu atributu Je aktivní na TRUE,
    4. nastaví hodnotu atributu Datum vytvoření na aktuální čas (čas uložení do databáze),
    5. nastaví hodnotu atributu Vytvořil na Uživatele, který spustil UC,
    6. nastaví hodnotu atributu Zdroj vytvoření dle Zdroje změny získaného na vstupu (vstupní parametr Zdroj změny).
  3. Systém vytvoří novou instanci entity Záznam o modifikaci instance – spustí 305UC03: Založit záznam o modifikaci instance, na vstup jsou předány následující informace:
    • Zakládaný záznam o modifikaci instance: předány jsou následující hodnoty atributů entity Záznam o modifikaci instance:
      • Typ entity: Fotografie Stanoviště (viz Typ entity),
      • Identifikace instance: identifikační číslo založené Fotografie stanoviště (viz krok 2),
      • Provedená akce: Založení (viz Akce při modifikaci instance).
    • Zdroj změny: Zdroj změny získaný na vstupu (vstupní parametr Zdroj změny).
  4. Pokud je na vstupu získán Záznam o pasportizaci nádoby (vstupní parametr Záznam o pasportizaci nádoby):
    1. Systém přiřadí vytvořený Záznam o modifikaci instance k tomuto Záznamu o pasportizaci nádoby – spustí 305UC04: Založit záznam o modifikaci instance při pasportizaci nádoby, na vstup jsou předány následující informace:
      • Zakládaný záznam o modifikaci instance při pasportizaci nádoby: předány jsou hodnoty jednotlivých atributů entity Záznam o modifikaci instance při pasportizaci nádoby:
        • Záznam o pasportizaci nádoby: na vstupu získaný Záznam o pasportizaci nádoby (vstupní parametr Záznam o pasportizaci nádoby).
        • Záznam o modifikaci instance: vytvořený Záznam o modifikaci instance (viz krok 3).
  5. Systém uloží provedené změny do databáze.
    • vytvořená instance entity Fotografie stanoviště,
    • vytvořená instance entity Záznam o modifikaci instance (předána na výstupu UC – výstupní parametr Záznam o modifikaci instance),
    • vytvořená instance entity Záznam o modifikaci instance při pasportizaci nádoby (pokud došlo k založení).

Validace

VAL01: Kontrola vstupních dat

V první řadě je provedena kontrola instance entity Záznam o pasportizaci nádoby (vstupní parametr Záznam o pasportizaci nádoby), která je získána na vstupu. Provedeny jsou následující akce:

  • kontrola, zda odkazovaná instance existuje (existuje instance se zadaným identifikátorem).

Dále je provedena kontrola vstupních dat vzhledem k jednotlivým atributům entity Fotografie stanoviště (vstupní parametr Zakládaná fotografie stanoviště), prováděny jsou pak následující akce:

  • kontrola datového typu (hodnota je požadovaného datového typu),
  • kontrola rozsahu hodnoty (hodnota se pohybuje v povoleném rozsahu),
  • kontrola povinnosti zadání hodnoty (povinné údaje jsou vyplněny),
  • kontrola existence odkazované hodnoty (zachování referenční integrity),
  • kontrola jedinečnosti v rámci tabulky (hodnota atributu, či kombinace atributů, se nesmí v tabulce opakovat).

Poznámka: Některé atributy neumožňují provedení všech akcí (např. kontrola existence odkazované hodnoty pro atribut obsahující prostý text).
