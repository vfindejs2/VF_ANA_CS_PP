Dokument popisuje technickú realizáciu riešenia UC zobrazenia zoznamu výsypov nádoby pre externého zákazníka. Realizácia sa odvíja na základe dokumentácie https://confluence.radium.cz/pages/viewpage.action?pageId=98238501.

Z dokumentácie použijeme základný popis:

Konceptuálna architektúra

Na celkové pochopenie problematiky znázorníme relačný model entít ktoré sa podieľajú na získanie týchto výsypov a následne ako REST API pracuje s týmito entitami.

 

Empty Container Event View (empty_container_event_view) - predstavuje databázový view ktorý prepojí jednotlivé výsypy nádob s priradeniami identifikátorov na nádoby (cez atribút “code”) a následne s priradeniami nádob na položky objednávok, pričom sa vyhodnotí z oboch prepojení časová platnosť a takisto aj čas výsypu na tieto časové okná.

 

Container External Customers Advanced Filters - definujú sa pre používateľa (externého zákazníka) filtre podľa ktorých pristupuje k nádobám. Detailný popis tu https://ai-maps.atlassian.net/wiki/spaces/FP/pages/5016485906/N+doby+-+Extern+z+kazn+ci#Definovanie-pokro%C4%8Dil%C3%BDch-filtrov

REST API

API pre výsypy nádob vyhodnotí či daný user má pokročilé filtre a tie aplikuje (ak existujú) na Empty Container Event View a následne pre konkrétnu nádobu získame výsypy - viď API https://ai-maps.atlassian.net/wiki/spaces/FP/pages/4962919185/Dokument+cia+API#V%C3%BDsypy-n%C3%A1dob---extern%C3%AD-z%C3%A1kazn%C3%ADci

Obdobná logika sa aplikuje aj na API pre získanie posledného výsypu nádob - viď API https://ai-maps.atlassian.net/wiki/spaces/FP/pages/4962919185/Dokument+cia+API#Posledn%C3%BD-v%C3%BDsyp-N%C3%A1doby----extern%C3%AD-z%C3%A1kazn%C3%ADci
